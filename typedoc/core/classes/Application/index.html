@webda/core • Readme | API

Class: Application

Map a Webda Application

It allows to:

  • Analyse imported modules
  • Scan code for Modda and generate the webda.config.json
  • Compile and Watch
  • Migrate from old configuration
  • List deployments

replaceVariables()​

replaceVariables(object, replacements): any

Allow variable inside object strings

Example

replaceVariables({
 myobj: "${test.replace}"
}, {
 test: {
   replace: 'plop'
 }
})

will return

{
 myobj: 'plop'
}

By default the replacements map contains

{
 git: GitInformation,
 package: 'package.json content',
 deployment: string,
 now: number,
 ...replacements
}

See: GitInformation

Parameters​

• object: any

a duplicated object with replacement done

• replacements: any= {}

additional replacements to run

Returns​

any
