# Pull Request Generator

## When to use

Use this skill to generate pull requests for code changes.

## Instructions

1. **Verify build status**: Ensure that the build is passing before generating a pull request running tests and checks locally. In the root of the project, run:

```
pnpm run build
pnpm run test
```

2. **Ensure code coverage**: Make sure that your code changes are covered by tests. You can check the code coverage report generated by your testing framework.

3. **Lint your code**: Run a linter to ensure that your code adheres to the project's coding standards. For example, if you're using ESLint, run:

```
pnpm run lint
```

4. **Generate pull request**
